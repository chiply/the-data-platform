# schema-registry

Centralized schema registry for the data platform

## Getting Started

### Prerequisites

- Python 3.12+
- Bazel (see `MODULE.bazel` at the monorepo root)
- PostgreSQL (for local development)

### Running Locally

```bash
uvicorn src.schema_registry.main:app --host 0.0.0.0 --port 8000 --reload
```

### Building with Bazel

```bash
# Build the service
bazel build //services/schema-registry/...

# Run all tests
bazel test //services/schema-registry/...
```

### Building with Docker

```bash
# Build the base image first (if not already built)
docker build -f monorepo/services/Dockerfile.base -t tdp-python-base monorepo/services/

# Build the service image
docker build -t schema-registry .
```

## Project Structure

```
src/schema_registry/
  __init__.py          # Package exports
  main.py              # FastAPI app factory
  config.py            # Pydantic settings
  schemas.py           # Pydantic models
  exceptions.py        # Exception hierarchy
  dependencies.py      # FastAPI dependency injection
  telemetry.py         # OpenTelemetry setup
  routers/
    health.py          # /health and /version endpoints
  database.py          # Async SQLAlchemy engine & session
  models/              # SQLAlchemy ORM models
tests/
  conftest.py          # Shared fixtures
  test_health.py       # Health endpoint tests
  test_schemas.py      # Schema round-trip tests
  test_properties.py   # Property-based tests
```

## Maintaining BUILD Files

The initial `BUILD.bazel` is generated by the Copier template as a starting point.
After generation, BUILD files are maintained by
[Gazelle](https://github.com/bazelbuild/rules_python/tree/main/gazelle)
(the `rules_python` Gazelle extension). Run Gazelle whenever dependencies change:

```bash
bazel run //:gazelle
```

Do not manually edit `BUILD.bazel` for dependency changes -- let Gazelle manage them.

## Post-Generation Checklist

After generating this service from the template, complete these steps:

- [ ] Create Helm chart at `deploy/charts/schema-registry/` (copy from `deploy/charts/common/` and customize)
- [ ] Register in `release-please-config.json` and `.release-please-manifest.json`
- [ ] Add service to Tiltfile for local development
- [ ] Run `bazel run //:gazelle` to sync BUILD files with actual dependencies
- [ ] Create the database and run initial Alembic migration
